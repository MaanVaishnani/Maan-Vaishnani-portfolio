IPL Data Analytics Pipeline

Python JSON Processing PostgreSQL Power BI Groq LLaMA3 Streamlit
View on GitHub ↗

A full-scale cricket analytics platform covering data ingestion, SQL analytics, Power BI dashboards, and an AI-powered chatbot — all deployed on Streamlit Cloud with a Supabase PostgreSQL backend.

What I built

  • Collected and processed 76,000+ IPL ball-by-ball records across 10+ seasons in JSON format and stored them in a normalized PostgreSQL (Supabase) database.
  • Performed advanced SQL analysis using window functions, CTEs, and subqueries to generate 10+ player performance KPIs and team insights.
  • Built a 5-page interactive Power BI dashboard with 15+ visualizations covering match trends, batting, and bowling analytics.
  • Developed and deployed an AI-powered chatbot using Groq LLaMA3 that converts natural language questions into SQL queries on Streamlit Cloud.

BRTS Transport Delay Analytics

PostgreSQL SQL Python Selenium Geopy Power BI DAX
View on GitHub ↗

A transport delay analytics platform for Ahmedabad's BRTS network, combining automated data collection, geospatial encoding, normalized SQL design, and Power BI dashboards for route-level decision-making.

What I built

  • Designed a normalized PostgreSQL database for 5+ entities — routes, stops, vehicles, trips, and passenger feedback.
  • Analyzed transport delays across 20+ routes using advanced SQL queries including joins, subqueries, and window functions to identify high-impact bottlenecks.
  • Automated route data collection using Python (Selenium) and geocoded 50+ stop locations using Geopy (OpenStreetMap).
  • Built an interactive Power BI dashboard with DAX measures to visualize route performance, delay trends, and passenger feedback KPIs.
